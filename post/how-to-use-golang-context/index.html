Golang에서 Context 사용하기

June 15, 2018 IT

Go에는 다른 대중적인 언어와 다른 컨셉들이 좀 있다.

  1. 클래스를 과감히 빼버렸고 그래서 상속이 없다
  2. Exception이란 것도 없다 예외상항 자체를 허용하지 않겠다는 의지인가? 멋있어 보일진 몰라도 솔직히 불편하다ㅠㅠ 궁시렁 궁시렁…
  3. 고루틴과 채널을 이용한 병행처리 모델도 친숙한 개념은 아니다

여기에 한가지 더 보태자면, Context란 녀석이다.

위 세가지는 Go 언어를 사용해서 뭔가를 만들려면 반드시 알아야 할 개념이기 때문에 Go 언어를 처음 접하는 대부분의 사람들이 시간을 할애해서 이 부분에 대해 공부를 한다. 하지만 컨텍스트는 사실 사용하지 않아도 로직을 구현하는데는 별 문제가 없어서 개념을 충분히 익히지 않은채 건너뛰는 경우가 많은 것 같다.

그래서 이번 글에서는 Context의 사용법을 소개한다.

컨텍스트란?

소프트웨어 공학에서 컨텍스트란 용어는 참 다양하게 사용된다.

  • context switching
  • bounded context
  • context menu
  • etc.

네이버 사전에서 찾아보면 맥락이라고 풀이되어 있다.

사전적 뜻을 그대로 가져와보면, 맥락을 유지하는 통로정도로 표현할 수 있고, Go에서의 실제 쓰임새를 봐도 이 의미와 크게 다르지 않다.

Go에서의 context.Context

이러한 맥락(=컨텍스트)을 유지하기 위해 Go는 context.Context 타입을 제공한다. context.Background 함수를 사용하여 컨텍스트를 생성하고, Context#Value 메서드를 사용하여 컨텍스트의 값을 가져온다. 컨텍스트에 값을 추가할때는 context.WithValue 함수로 새로운 컨텍스트를 만들어 주어야 한다.

	// 컨텍스트 생성
	ctx := context.Background()

	// 컨텍스트에 값 추가
	// context.WithValue 함수를 사용하여 새로운 컨텍스트를 생성함
	ctx = context.WithValue(ctx, "current_user", currentUser)

	// 함수 호출시 컨텍스트를 파라미터로 전달
	myFunc(ctx)
func myFunc(ctx context.Context) error {
	var currentUser User

	// 컨텍스트에서 값을 가져옴
	if v := ctx.Value("current_user"); v != nil {
		u, ok := v.(User)
		if !ok {
			return errors.New("Not authorized")
		}
		currentUser = u
	}

	// currentUser를 사용하여 로직 처리

	return nil
}

위와 같이 ctx 에 현재 맥락(=컨텍스트)안에서 유지해야 할 값들을 저장한 후 ctx를 계속 가지고 다니면서 컨텍스트 내의 값을 사용한다.

package context

type Context interface {
	Deadline() (deadline time.Time, ok bool)
	Done() <-chan struct{}
	Err() error
	Value(key interface{}) interface{}
}

컨텍스트 생성은 context.Background() 함수를 사용한다.

그리고 아래 함수를 통해 context.Context를 생성한다. 

func Background() Context
func WithCancel(parent Context) (ctx Context, cancel CancelFunc)
func WithDeadline(parent Context, d time.Time) (Context, CancelFunc)
func WithTimeout(parent Context, timeout time.Duration) (Context, CancelFunc)
func WithValue(parent Context, key, val interface{}) Context

맥락을 유지하는 역할

하나의 기능이 시작되어 마무리되는 일련의 흐름을 맥락(=컨텍스트)이라 표현할 수 있을 것 같다. 예를들면 웹 어플리케이션에서 사용자의 요청이 들어왔을때, 그 요청을 받아 처리해야 할 일련의 과정들을 수행한 후에 클라이언트에 response가 전달될때까지를 하나의 맥락이라 볼 수 있고, 이 맥락 안에서 유지되어야 할 값들이 있다면 그것을 context.Context에 담아두면 된다.

Go의 http.Request 타입은 아래와 같이 정의되어 있다. 

type Request struct {
	Method string	
	Header Header
	Body io.ReadCloser	
	ContentLength int64	
	Host string	
	RemoteAddr string
	RequestURI string	
	Response *Response

	/* ... */

	ctx context.Context
}
여기서 맨 마지막 필드로 ctx context.Context가 정의되어 있는데, 이 컨텍스트를 통해 웹 요청이 완료될때까지의 맥락을 유지한다.

고루틴의 생명주기 제어

Context가 정말 필요한 경우가 고루틴의 life cycle을 제어할때이다. Go 언어의 초기 버전에서는 context 패키지가 없었다. 그래서 각자 자기만의 방식대로 고루틴의 라이프사이클을 제어했었다. 고루틴을 사용할때 주의해야 할 사항 중 하나는 “고루틴이 언제 종료될지 명확하게 알고 사용하라”인데 컨텍스트가 없던 그 시절에는 이를 제어하기 위해 참 다양한 방법이 존재했었다.

하지만 1.x 버전에서 context 패키지가 들어오고, Go는 고루틴을 제어할 수 있는 일관된 방식을 가이드했다.

바로 deadline, canclation등의 방식이다.
